# CloudServe 技术架构决策

## 项目概述
云服务商 CloudServe 前端页面 + 控制台 + 自建后端

## 技术栈决策

| 项目 | 决策 | 备注 |
|------|------|------|
| 前端框架 | Vue 3 SPA (CDN) | 保持现有架构 |
| 控制台前端 | Vue 3 SPA | 新建 console/ 目录 |
| 后端框架 | Node.js + Express | 统一 JS 技术栈 |
| 数据库 | PostgreSQL | 用户确认 |
| ORM | Prisma | 类型安全、自动迁移 |
| 认证 | JWT + Refresh Token | 无状态 |
| 支付 | 微信支付 V3 + 支付宝当面付 | 先预留接口，商户号待申请 |
| 实时通信 | Socket.io | 预留，监控数据先用轮询 |
| 部署 | Docker Compose | 后端 + PostgreSQL 在 Docker |

## 架构原则
- **可迁移性优先**：所有业务逻辑、支付、监控接口预留，不写死
- **前后端分离**：API RESTful，接口文档先行
- **模拟数据**：监控数据用模拟数据，保证接口一致，后期可替换真实数据源

## 数据库
- PostgreSQL（Docker 内运行）
- Prisma ORM 管理迁移

## 支付
- 微信支付 V3 API - 接口预留
- 支付宝当面付 - 接口预留
- 商户号待申请
